# 软件系统测试用例

本组用例属于软件系统级测试，通过在前端界面上操作完成。

| 用例编号  | 测试目标                                                     | 前置条件                                                     | 操作步骤                                                     | 预期结果                                                     |
| --------- | ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| TC-SYS-01 | 验证已注册的用户可以登录账户                                 | 用户已注册                                                   | 在登录页面，输入正确的用户名、密码后，点击“登录”按钮或按下“Enter”键 | 显示“登录成功”提示信息，并进入登录用户的“全部任务”页面       |
| TC-SYS-02 | 验证密码错误时不可非法登录账户                               | 用户已注册                                                   | 在登录页面，输入正确的用户名与错误的密码后，点击“登录”按钮或按下“Enter”键 | 显示“密码错误”提示信息，页面不切换                           |
| TC-SYS-03 | 验证可以创建任务，且可以查看正确的任务详情                   | 用户已登录                                                   | 在“个人任务”页面，点击“新建个人任务”按钮，完成初始设置并点击“确认”按钮后，点击当前页面的任务列表中该任务的查看任务详情按钮 | 任务详情页上可以看到正确的初始设置，且刷新页面或重新登录后仍能在“全部任务”与“个人任务”页面的任务列表中看到该任务 |
| TC-SYS-04 | 验证可以删除任务                                             | 用户已登录                                                   | 在含任务列表的页面中，点击任务列表中特定任务的删除任务按钮   | 当前页面的任务列表中不再能看到该任务，且刷新页面或重新登录后在相同页面上的任务列表中仍不能看到该任务 |
| TC-SYS-05 | 验证可以创建团队                                             | 用户已登录                                                   | 在“全部团队”或“我拥有的团队”页面，点击“新建团队”按钮，在弹出的窗口中设置团队名并点击“确认”按钮 | “全部团队”与“我拥有的团队”页面的团队列表中呈现该团队，且卡片中呈现的“拥有者”为当前用户的用户名，“我的角色”为“拥有者”，“人数”为 1 |
| TC-SYS-06 | 验证 owner 权限的用户可以添加成员                            | 用户已登录，团队已创建，且用户的权限为 owner                 | 在团队空间页面，点击“编辑成员信息”按钮进入团队成员页面，并点击“添加成员”按钮，在弹出的窗口中输入另一个已经注册的用户的用户名，点击“确认”按钮，随后登录另一个用户的账户，点击通知中邀请加入团队消息的“接受”按钮，并刷新原用户的界面 | 原用户的团队列表中该团队的卡片中呈现的“人数”增加了 1，编辑成员信息界面上出现了另一个用户的用户名，且另一个用户的“全部团队”与“我参与的团队”页面上的团队列表中可以看到该团队，包括自己的权限“参与者” |
| TC-SYS-07 | 验证 owner 权限的用户可以编辑团队成员权限                    | 用户已登录，团队已创建，用户的权限为 owner，且团队中至少有一个 member 权限的成员 | 在团队空间页面，点击“编辑成员信息”按钮进入团队成员页面，并将某个 member 权限成员的用户名从“参与者”栏拖拽到“管理者”栏 | 被更改权限的团队成员登录账户后收到关于其权限变化的通知，并可以在“我管理的团队”页面看到该团队，但不再能在“我参与的团队”页面看到该团队，团队列表中该团队卡片上该用户的权限变为”管理者“，且团队空间的任务列表中所有任务都显示编辑任务和删除任务按钮 |
| TC-SYS-08 | 验证 admin 权限的用户可以分配任务负责人                      | 用户已登录，团队已创建，且用户的权限为 admin                 | 在团队空间页面，点击任务列表中某一个任务的编辑任务按钮，并在编辑团队任务页面修改负责人，最后点击“确认”按钮 | 任务列表中当前编辑的任务的负责人变更为新的用户名，登录新负责人的账户后可以看到关于被分配新的团队任务的通知信息，并且切换到该团队的团队空间页面后可以看到任务列表中该任务的操作栏中出现开始任务（任务状态：新建）或完成任务（任务状态：进行中）按钮 |
| TC-SYS-09 | 验证可以添加任务依赖关系                                     | 用户已登录，至少有两条个人任务 A, B                          | 在“全部任务”或“个人任务”页面，点击任务列表中任务 B 的编辑任务按钮，并在编辑任务页面上点击编辑前置任务按钮，在弹出的窗口中选中任务 A 的标题，点击窗口中的“确认”按钮，并点击编辑任务页面上的“确认”按钮，回到团队空间页面，分别点击任务 A 与任务 B 的查看任务详情按钮 | 任务 A 的详情页面上的“后继任务”中出现了任务 B 的标题，任务 B 的详情页面上的“前置任务”中出现了任务 A 的标题 |
| TC-SYS-10 | 验证任务依赖关系不能出现环路                                 | 用户已登录，至少有三条个人任务 A, B, C，且有依赖关系 B 依赖 A，C 依赖 B | 在“全部任务”或“个人任务”页面，点击任务列表中任务 A 的编辑任务按钮，并在编辑任务页面上点击编辑前置任务按钮，在弹出的窗口中选中任务 C 的标题，尝试点击窗口的“确认”按钮，随后点击窗口的“取消”按钮，并点击编辑任务页面上的“确认”按钮，回到团队空间页面，点击任务 A 的查看任务详情按钮 | 尝试点击窗口的“确认”按钮后显示产生依赖关系环路的提示信息，且窗口不关闭；在任务 A 的详情页面上的“前置任务”中不出现任务 C 的标题 |
| TC-SYS-11 | 验证可以通过完成任务按钮更新任务状态，任务存在未完成的前置任务时不能被完成，且任务依赖关系可以被删除 | 用户已登录，至少有两条状态为“进行中”的个人任务 A, B，且有依赖关系 B 依赖 A | 在“全部任务”或“个人任务”页面，尝试点击任务列表中任务 B 的完成任务按钮，随后点击任务 B 的编辑任务按钮，并在编辑任务页面上点击编辑前置任务按钮，在弹出的窗口中取消选中任务 A 的标题，点击窗口的“确认”按钮，并点击编辑任务页面上的“确认”按钮，回到团队空间页面，再次尝试点击任务 B 的完成任务按钮，并点击任务 B 的查看任务详情按钮 | 第一次尝试点击任务 B 的完成任务按钮时显示任务 A 这个前置任务未完成，无法完成任务 B 的提示信息；第二次尝试点击任务 B 的完成任务按钮时显示“任务已完成”的提示信息，完成任务按钮消失，且在任务 B 的任务详情页面上“状态”为“已完成”，前置任务中无任务 A 的标题 |
| TC-SYS-12 | 验证非 owner 权限的团队成员可以退出团队，且退出团队的成员负责的任务可以妥善处理 | 用户已登录，团队已创建，用户的权限为 owner，且团队中除了 owner 外至少还有一个成员，且该成员至少有一条未完成的任务与一条已完成的任务 | 登录团队中另一个成员的账户，切换到“全部团队”或“我管理/参与的团队”页面，点击该团队的卡片右上角的更多按钮，并点击“退出团队”选项 | 另一个成员含团队列表的页面上不再呈现该团队的卡片，且该团队 owner 的账户上刷新页面后收到通知，告知 owner 某个成员主动退出团队，其负责的未完成的任务自动转交给 owner，同时团队空间的任务列表中该退出成员负责的已完成的任务的负责人栏处添加了“（已离队）”标签 |
| TC-SYS-13 | 验证 owner 权限的用户可以解散团队                            | 用户已登录，团队已创建，用户的权限为 owner，且团队中除了 owner 外至少还有一个成员 | 在“全部团队”或“我拥有的团队”页面，点击该团队卡片右上角的更多按钮，并点击“解散团队”选项，最后点击“回收站”按钮 | 所有该团队的成员不再能在含团队列表的页面上看到该团队，当前用户在“回收站”页面上已解散的团队列表中可以看到该团队的信息，且其他该团队的成员在登录账户后收到关于该团队已解散的通知 |
| TC-SYS-14 | 验证 owner 权限的用户可以转让 owner 权限                     | 用户已登录，团队已创建，用户权限为 owner，且团队中除了 owner 外至少还有一个成员 | 在“全部团队”或“我拥有的团队”页面，点击该团队卡片右上角的更多按钮，并点击“转让拥有者权限”选项，在弹出的窗口中选择希望转让的对象，点击“确认”按钮，随后登录转让对象的账户，点击通知中请求转让拥有者权限消息的“接受”按钮，并刷新原用户的界面 | 原用户含团队列表的界面中该团队的卡片上显示的“我的角色”变成“管理者”，而接受转让 owner 权限的用户的团队列表中该团队的卡片上显示的“我的角色”变成“拥有者” |
| TC-SYS-15 | 验证任务任务的新建、编辑、删除操作都按时间倒序记录在日志中   | 用户已登录                                                   | 在“个人任务”页面，点击“新建个人任务”按钮，完成初始设置并点击“确认”按钮，随后在任务列表中点击该任务的编辑任务按钮，对该任务的两个个不同属性进行修改后点击“确认”按钮，最后在任务列表中点击该任务的删除按钮，并点击“查看操作日志”按钮 | 个人任务操作日志页面上的表格中按时间倒序显示四条该任务的记录，最早的记录是新建任务，随后是两条同时发生的编辑任务记录，最后是删除任务记录 |

以上测试全部通过。
